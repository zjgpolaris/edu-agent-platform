# v1.25 试点主路径开发记录

## 背景

截至 v1.24，EduAgent 已经打通学生端今日计划、错题本、复习、AutoTutor，以及教师端待复核、作业完成情况、班级学情、共性错题和命题质量看板等能力。下一阶段需要从“功能丰富”收敛到“可稳定演示、可小范围试点”的主路径。

v1.25 的目标是让系统具备一套可重复准备的 pilot demo 数据，并让学生/教师首页明确回答：

- 学生：今天下一步该学什么，为什么？
- 教师：今天先处理什么教学动作？

## 本次改动范围

### 1. Pilot demo seed

新增 `scripts/seed_pilot_demo.py`，用于灌入一套固定试点数据。

默认账号：

| 角色 | 账号 | 密码 | 用途 |
|------|------|------|------|
| 教师 | `pilot-teacher` | `pilot123` | 查看教师工作台、质检盲区、作业完成情况 |
| 学生 | `pilot-student` | `pilot123` | 查看学生今日计划、错题复习、AutoTutor 入口 |
| 学生 | `pilot-student-b` | `pilot123` | 辅助制造提交/评阅样本 |
| 学生 | `pilot-student-c` | `pilot123` | 辅助制造共性错题样本 |
| 学生 | `pilot-student-d` | `pilot123` | 辅助制造欠交/逾期样本 |

Seed 内容：

- 创建或重置 pilot 账号密码。
- 创建/复用一份 `【Pilot Demo】辛亥革命随堂诊断` 作业。
- 三名学生提交作业，其中同一道 `quality.level = "ok"` 的客观题多人答错，触发质检盲区。
- 保留 `pilot-student` 的 partial 提交，触发教师待复核。
- 评阅 `pilot-student-b`，制造已评阅样本。
- `pilot-student-d` 不提交，制造欠交/逾期。
- 为 `pilot-student` 写入 weakpoints。
- 为 `pilot-student` 写入今日 review placeholder，使用 `pending_generate`，不触发 LLM。
- 写入一条未读通知，并通过查重保证重复 seed 不膨胀。

运行方式：

```bash
PYTHONPATH=backend python3 scripts/seed_pilot_demo.py
```

如果后端通过 `.env.local` 指向特定数据库，应使用同一环境运行：

```bash
set -a; [ -f .env.local ] && . ./.env.local; set +a; PYTHONPATH=backend python3 scripts/seed_pilot_demo.py
```

### 2. Pilot path smoke

新增 `eval/pilot_path_smoke.py`，并注册到：

- `eval/run_core_evals.py`
- `Makefile eval-smoke`

覆盖 6 个 case：

1. `seed_is_idempotent`：seed 重复执行不重复作业/通知。
2. `pilot_accounts_authenticate`：pilot 教师/学生账号可认证。
3. `student_today_plan_has_next_step_without_llm`：学生今日计划有下一步任务，weakpoint task 指向 AutoTutor focus 链接，不触发 LLM。
4. `teacher_queue_backend_signals_exist`：教师待复核、欠交/逾期、质检盲区信号存在。
5. `notification_banner_seeded_once`：通知只生成一条未读。
6. `weakpoints_and_review_are_connected`：weakpoints 与 review placeholder 已连接，读取时使用 `hydrate=False`。

### 3. 学生下一步学习卡

- `frontend/app/(student)/student/useStudentWorkbenchData.ts`
- `frontend/app/(student)/student/ContinueLearningCard.tsx`

### 4. 教师今日教学队列

- `frontend/app/(teacher)/teacher/TeacherTodayQueue.tsx`

### 5. 登录页一键体验

## 使用方式

### 准备数据

```bash
set -a; [ -f .env.local ] && . ./.env.local; set +a; PYTHONPATH=backend python3 scripts/seed_pilot_demo.py
```

### 学生路径

1. 打开 `http://localhost:3000`。
2. 保持「学生」身份。
3. 点击「Pilot 学生A · 学生体验」。
4. 进入 `/student`。
5. 检查继续学习卡、今日计划、通知横幅、AutoTutor focus 入口。

### 教师路径

1. 打开 `http://localhost:3000`。
2. 切换到「教师」身份。
3. 点击「Pilot 张老师 · 教师体验」。
4. 进入 `/teacher`。
5. 检查今日教学队列中的待复核、质检盲区、欠交/逾期等行动项。
6. 进入 `/teacher/quality-dashboard` 查看质检盲区详情。

## 设计约束

- 首页只读聚合，不自动触发 LLM。
- 学生卡片只消费 `/today`，不自行发明推荐规则。
- 教师队列只读聚合，不自动生成讲评稿。
- Seed 尽量使用现有 service API，少量账号密码重置和 review placeholder 使用 SQL fallback。
- Pilot smoke 使用 isolated SQLite，可离线运行。

## 后续建议

1. 作业结果页错题行动增强：学生提交作业后，直接提供 AutoTutor focus 入口。
2. 教师端待复核数字口径继续与侧边栏 badges 对齐。
3. 若首页接口变多导致加载慢，可后续新增 `/api/teacher/today-queue` 后端聚合接口。
4. v1.26 可继续推进 AutoTutor exit ticket 与辅导效果证据闭环。
